# Prettier

安装 Prettier:

npm i -D prettier

在项目根目录添加 Prettier 的配置 .pretterrc.js:

module.exports = {
  overrides: [
    {
      files: "*.{js,jsx,ts,tsx,vue}",
      options: {
        singleQuote: true,
        printWidth: 100
      }
    }
  ]
}

在 package.json 中添加用于格式化代码的脚本:

"scripts": {
  "lint": "prettier --write resources/**/*.{js,ts,jsx,tsx,vue,css,scss,sass,json}"
},

# 为项目添加 favicon

理论上我们不应该直接操作 public 文件夹下的任何内容，因此我们在 webpack.mix.js 中添加以下配置:

mix
  // ...
  .copy('resources/assets/favicon.png', 'public/favicon.png');

在 resources/views/layouts/app.blade.php 的 <head> 标签中引入 favicon:

<head>
  <!-- ... -->
  <link rel="shortcut icon" href="{{ asset('favicon.png') }}">
  <!-- ... -->
</head>

# 支持静态资源文件夹

事实上总有些资源是不可能被 Webpack 完美处理的，因此我们需要将部分资源放置到 public 文件夹下直接使用。我们依然可以通过 Laravel-Mix 来帮助我们处理这些资源。

在 webpack.mix.js 中添加以下配置:

mix
  // ...
  .copyDirectory('resources/static', 'public/static');

在项目根目录创建 .prettierignore，忽略静态资源文件夹:

resources/static

创建 resources/static/.gitkeep。

# 在生产环境下为资源添加版本号

在 webpack.mix.js 中添加配置:

if (mix.inProduction()) mix.version();

修改 resources/views/layouts/app.blade.php 中的 assets() 函数为 mix()

<!doctype html>
<html lang="{{ str_replace('_', '-', app()->getLocale()) }}">
<head>
  <!-- ... -->

  <link rel="shortcut icon" href="{{ mix('favicon.png') }}">

  <!-- Scripts -->
  <script src="{{ mix('js/bootstrap.js') }}"></script>

  <!-- Styles -->
  <link href="{{ mix('css/app.css') }}" rel="stylesheet">
</head>
<body>
<!-- ... -->
<script src="{{ mix('js/app.js') }}"></script>
@yield('scripts')
@stack('scripts')
</body>
</html>
